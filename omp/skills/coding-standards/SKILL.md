# Rules

- KISS, DRY, YAGNI, SOLID; enforce strictly
- Self-documenting code; no inline comments
- Top-to-bottom reading order, code should read like a story
- Guard clauses + early returns; NEVER nest positive conditional logic
- Prefer pure functions, immutability, composition
- No premature abstraction; extract only on second use
